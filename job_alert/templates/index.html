Email Setup

Step 1

Pick the Gmail account that will send the alerts

Use the Gmail account that should send the job-alert emails. Best practice is a dedicated sender account, while your normal inbox only receives the alerts.

What this means in plain language

This tool does not send email on its own. It logs into one Gmail account and asks Gmail to send the alert for you. That Gmail account is the sender.

If you use a separate sender Gmail, the alerts arrive more cleanly and stay separate from your personal mail activity.

Step 2

Turn on 2-Step Verification in that Gmail account

Google blocks normal password login for apps like this. 2-Step Verification must be enabled before Google allows App Passwords.

Exactly where to click

Open the sender Gmail account in your browser, then go to Google Account > Security > 2-Step Verification.

If it says 2-Step Verification is off, turn it on and finish Google's setup flow first.

Open Google Security Settings

Step 3

Create a Gmail App Password

This is the most important step. The field below does not want your normal Gmail password. It wants a special Google-generated App Password.

What an App Password is

An App Password is a special one-time password Google creates for apps that cannot do the normal browser login flow. It is separate from your real Gmail password.

Google usually shows it as a 16-character code, sometimes grouped with spaces. That is normal.

How to get it

While logged into the sender Gmail account, open Google Account > Security > App Passwords.

Create a new App Password, copy the code Google shows you, and paste that code into the `Gmail App Password` field below.

Open Google App Passwords

If you do not see the App Passwords page

That usually means 2-Step Verification is not fully enabled yet, or this Google account has a stricter security mode such as Advanced Protection.

Step 4

Fill in the fields below exactly like this

Field-by-field guide

Sender Email: the Gmail account that sends the alerts. Example: `job.search.updates.max@gmail.com`.

Sender Name: the friendly name shown in the email. Example: `Job Alert`.

SMTP Username: for Gmail, this should normally be the exact same Gmail address as `Sender Email`.

Recipient Emails: the inbox that should receive the alerts. This can be your personal Gmail.

Gmail App Password: paste the Google-generated App Password here. Do not paste your normal Gmail password.

Step 5

Save and test immediately

Click `Save + Send Test Email`. If it works, the email part of setup is finished. If it fails, expand the troubleshooting help below before changing random fields.

Common mistakes that cause failure

Using the normal Gmail password instead of an App Password.

Using an App Password created for a different Google account.

Typing a different Gmail address in `Sender Email` and `SMTP Username`.

Trying to use App Passwords before 2-Step Verification is fully enabled.

Advanced SMTP server settings

Most users should leave these on the Gmail defaults shown below.

Quick troubleshooting if the test still fails

If Gmail says `Application-specific password required`, you almost certainly entered the normal Gmail password instead of the Google App Password.

If Gmail says the username or password is wrong, create a fresh App Password in the exact sender account and paste it again.

If `App Passwords` is missing in your Google account, finish 2-Step Verification first and then check again.

Windows Schedule

  1. Choose how often the scraper should run.
  2. Set the time for when your PC is usually on.
  3. Click `Save + Apply Windows Schedule` so the scheduler is created immediately.

Why this is needed

Without Windows Task Scheduler, nothing runs automatically. The schedule is what turns this from a manual tool into a daily job alert service on your PC.

First Run

  1. After email and schedule are ready, run the scraper once from here.
  2. This first run stores the current job state so the app can distinguish future new jobs from already-seen ones.
  3. After that, use the `Job Alerts` tab and the `Check Current Matches` button to inspect sites live.
What happens on the first run

The first run boots the database with the current jobs. After that, the tool can tell what is genuinely new and avoid reminding you about the same posting every day.
