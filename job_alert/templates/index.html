Setup Wizard

Follow these steps in order. This wizard tells you what each setting is for and where to fill it in.

Step 1: Choose the sender email account

The app cannot send alerts by itself. It needs one mailbox to hand the alert to, and that mailbox sends the email for you.

The cleanest setup is a dedicated Gmail account just for alerts, with your normal inbox as the recipient.

Step 2: Create a Gmail App Password

Google does not want this app to use your main Gmail password. Instead, turn on 2-Step Verification for the sender account and create an App Password.

Paste that App Password into the Gmail App Password field below. That is what lets the tool send alerts safely.

Step 3: Save and send a test email

After filling in the sender email, SMTP username, app password, and recipients, save the setup and click Send Test Email.

If the test email works, the alert delivery is ready.

Step 4: Choose how often it should run

Set the schedule mode, repeat interval, time, and weekdays if needed. Then click Apply Windows Schedule.

That creates the Windows Task Scheduler job so the scraper keeps running without you manually starting it every day.

Step 5: Run it once and check diagnostics

Run the scraper once so the app can discover matching jobs and record the current site status.

After that, the Diagnostics tab will show you whether each site succeeded, failed, or needs attention.

Setup Details

Email Delivery

These fields tell the app which mailbox sends the alert and which inbox receives it.

Why this is needed

The app does not own an email service. It logs into one sender account, hands Gmail the alert, and Gmail sends the message to your recipient inboxes.

Automatic Schedule

These fields describe how often the scraper should run. After saving, click Apply Windows Schedule below.

Why this is needed

Without a Windows scheduled task, the scraper only runs when you launch it manually. The scheduled task is what makes it check jobs on its own.

How Matching Works

The scraper uses plain text matching, not AI interpretation during runtime.

What this means

Your `include_any` terms should cover the titles and wording you want. Your `exclude_any` terms should only block roles you are truly sure you never want. Location, contract type, and language lists help narrow results without needing hardcoded logic.
