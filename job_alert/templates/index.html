Setup Wizard

This is the foolproof walkthrough. Enter the required information directly here, save it for real, then move to the next topic at the bottom.

Email Setup

  1. Create or choose the Gmail account that will send the alerts. A dedicated alert Gmail is the cleanest setup.
  2. In that Gmail account, turn on 2-Step Verification and create a Gmail App Password for this tool.
  3. Enter the account details below and click `Save + Send Test Email` to confirm it works immediately.
Advanced SMTP server settings
Why each email field exists

`Sender Email` is the mailbox Gmail sends from. `SMTP Username` is usually the same Gmail address. `Gmail App Password` is the special password Google gives apps. `Recipients` are the inboxes that receive the alerts.

Windows Schedule

  1. Choose how often the scraper should run.
  2. Set the time for when your PC is usually on.
  3. Click `Save + Apply Windows Schedule` so the scheduler is created immediately.
{{ "installed" if scheduler_status.exists else "not installed" }} Next run: {{ scheduler_status.next_run_time or "Not scheduled yet" }}
Why this is needed

Without Windows Task Scheduler, nothing runs automatically. The schedule is what turns this from a manual tool into a daily job alert service on your PC.

First Run

  1. After email and schedule are ready, run the scraper once from here.
  2. This first run stores the current job state so the app can distinguish future new jobs from already-seen ones.
  3. After that, use the `Job Alerts` tab and the `Check Current Matches` button to inspect sites live.
What happens on the first run

The first run boots the database with the current jobs. After that, the tool can tell what is genuinely new and avoid reminding you about the same posting every day.

Email Delivery

These fields tell the app which mailbox sends the alert and which inbox receives it.

Why this is needed

The app does not own an email service. It logs into one sender account, hands Gmail the alert, and Gmail sends the message to your recipient inboxes.

Automatic Schedule

These fields describe how often the scraper should run. After saving, click Apply Windows Schedule below.

Why this is needed

Without a Windows scheduled task, the scraper only runs when you launch it manually. The scheduled task is what makes it check jobs on its own.

Default Profile

How Matching Works

The scraper uses plain text matching, not AI interpretation during runtime.

What this means

Your `include_any` terms should cover the titles and wording you want. Your `exclude_any` terms should only block roles you are truly sure you never want. Location, contract type, and language lists help narrow results without needing hardcoded logic.
